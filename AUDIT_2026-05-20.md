## Overall verdict

Solid foundation, dangerous gaps on the edges.

**What is correct**:
- Double-entry balance enforced on both TX-currency and base-currency axes
- Invoice / PayBill / ReceivePayment main paths wrapped in `IUnitOfWork.ExecuteAsync`
- Settlement idempotency + `SELECT FOR UPDATE` locking
- UnitySearch permission filter reads `company_user_permissions` fresh at SQL layer (revocation is immediate)
- Owner uniqueness enforced at DB level (`uq_company_memberships_one_owner`)
- SysAdmin vs Business APIs fully isolated (separate session stores, separate tokens)

**What is broken**:
- **Reverse flow is non-atomic** — three independent transactions, half-commit risk on failure
- **Sales-issue COGS is NOT reversed on invoice reverse** — inventory subsidiary ledger drifts from GL forever after every reverse
- **Inventory Adjustment / Write-off / Transfer / Manufacturing post NO GL entries** — only `inventory_ledger_entries`, no `journal_entries`. Inventory qty × cost cannot reconcile to GL Inventory Asset.
- **Inventory qty race**: no `FOR UPDATE`, no `CHECK (on_hand_qty >= 0)`, no advisory lock — two concurrent issues can both pass the negative-stock guard
- **Massive backend permission gap**: Payment Post, all `save-and-post` composites, Goods Receipt Post, FX Revaluation Post, Posted JE Void, Invoice Email Send, Customer/Vendor/Account/Item/Warehouse/TaxCode CRUD, Report Exports — **all ungated**. Catalog tokens exist, just not wired.
- **Dual permission stores diverge** — `company_user_permissions` (new) vs `company_memberships.permissions` JSONB (legacy) — 4F UI writes only new table, SysAdmin Preset Apply writes only JSONB, three different decision paths use different sources
- **Unauthenticated internal AI endpoint** accepts arbitrary `companyId`

## Confirmed-correct areas (don't break these)

- Double-entry balance enforced in both TX and base currency by `DefaultJournalAggregator.EnsureJournalInvariants` + `PostgresJournalEntryWriter.EnsureDraftIsBalanced`
- Invoice / Bill / Payment main POSTs wrapped in `IUnitOfWork.ExecuteAsync`
- Settlement: `HasExistingApplicationsAsync` idempotency short-circuit + `SELECT FOR UPDATE` on target open item
- Posted-document Void blocked by `void_document` policy (returns `blocked_by_policy`, no state mutation)
- Reverse compensating JE swaps Dr/Cr atomically, idempotent via `TryFindExistingCompensationAsync`
- Task billing coordinator's "already billed by different invoice" reject path
- Task state CAS guards (`where status = @from_status`) + transitions audit in same tx
- Task API endpoints all carry `RequireModuleEnabled` + `RequirePermission`
- H-4 FK migration well-formed (orphan pre-flight, idempotent DO blocks, RESTRICT DEFERRABLE)
- UnitySearch SQL-level permission filtering — revocation immediate, not session-cache delayed
- Owner uniqueness: DB-level `uq_company_memberships_one_owner` unique index + workflow-level `FOR UPDATE` count
- Owner transfer atomicity: same-tx SELECT FOR UPDATE + UPDATE + audit
- Permission grant/revoke audit in same transaction as state change
- SysAdmin vs Business API fully isolated (separate session stores, tokens, route trees)
- `BusinessRequestContractGuard` reflects over request bodies to reject CompanyId/UserId mismatches
- Generic forgot-password response (no user enumeration via timing/shape)
- Repository layer uniformly scopes by `company_id`
