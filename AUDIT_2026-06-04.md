## A. Overall verdict

The foundation is sturdier than feared; debt is highly concentrated. Not a
"big ball of mud": accounting truth is overwhelmingly server-authoritative,
dead code is ~zero, the design system is mature and dark-mode-ready, and
debit==credit is enforced at three production layers. The real debt is
concentrated in a few mega-files, copy-paste create pages, one raw-SQL UI
breach, a total absence of virtualization/server-paging, and missing direct
tests on core posting.

**Top 5 problems:** (1) `Program.cs` 16,316-line god file; (2) copy-paste
create pages (caused the real Bill tax=0 regression); (3) zero

fields (P-04) **and** core posting has no direct balance test (P-20) → a
tax/posting bug can ship undetected. This already happened once. Close it with
Batch 4 defensive tests first.
